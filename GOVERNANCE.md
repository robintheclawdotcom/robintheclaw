# Governance

Robin the Claw uses a maintainer-led model.

## Decisions

Changes are proposed as pull requests and reviewed against repository policy, code ownership
(`.github/CODEOWNERS`), and CI. Maintainers make the final call on what merges. Protocol, security,
and release decisions rest with the maintainers.

## Roles

- **Contributors** open issues and pull requests.
- **Maintainers** review, merge, cut releases, and steward security. Listed in `MAINTAINERS.md`.

## Changes to governance

This document changes by pull request with maintainer approval. Material changes are noted in the
changelog.
